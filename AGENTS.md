# AGENTS.md

## いずれの場合も含め、以下の要素を遵守し、実装を進めてください。
1. コメントでは「何をするか」ではなく「何をするため」を記入してください。
2. placeholderや将来的に値が存在することが明記されているものについては、`#TODO`を利用し、指定されたissue idを必ず記入してください。
3. コードに過度な型付けは不要です。-> NoneやUnion、typingによるList, Tupleなどは利用せず、list[..], str | int, Optional[str] など簡易的なもののみを利用してください。
4. ドキュメントを作成する場合、必ず /docs/summary/.. に配置してください。
5. issueの初期化による作業の場合、指定されたファイル以外は触らないでください。
6. indentにはスペースを利用し、2文字幅で実装してください。
